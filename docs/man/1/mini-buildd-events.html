    MINI-BUILDD-EVENTS(1) User Commands MINI-BUILDD-EVENTS(1)

    NAME

    mini-buildd-events - Monitor events. Optionally exit when criteria is met.

    DESCRIPTION

    usage: mini-buildd-events [-h] [--version]

    [-l {CRITICAL,ERROR,WARNING,INFO,DEBUG}] [-v] [-q]
    [-A AFTER] [-T {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...]] [-D DISTRIBUTION] [-S SOURCE] [-V SOURCE_VERSION] [-M MINIMAL_VERSION] [-E {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...]] [-F {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...]] ENDPOINT

    Monitor events. Optionally exit when criteria is met.

    positional arguments:

    ENDPOINT
    HTTP target endpoint: Twisted-style Network Client Endpoint
    See: https://twistedmatrix.com/documents/current/core/howto/endpoints.html#clients
    Notation Examples::
    'tcp:host=localhost:port=8066': Connect to localhost, unencrypted.
    'tls:host=localhost:port=8066': Connect to localhost, encrypted. 'http://localhost:8066' : Extra URL-style syntax (prefer twisted-style in saved configs, though). 'localhost:8066' : Compatibility host:port syntax (don't use -- might eventually go away).

    options:

    -h, --help
    show this help message and exit
    --version
    show program's version number and exit
    -l {CRITICAL,ERROR,WARNING,INFO,DEBUG}, --log-level {CRITICAL,ERROR,WARNING,INFO,DEBUG}
    set log level (DEBUG will enable exception tracebacks and python warnings) (default: INFO)
    -v, --verbose
    DEPRECATED (use --log-level): increase log level. Give twice for max logs (default: 0)
    -q, --quiet
    DEPRECATED (use --log-level): decrease log level. Give twice for min logs (default: 0)
    -A AFTER, --after AFTER
    Also show past events after this date. Use keyword 'epoch' for a big-bang timestamp. Otherwise most formats should work (like '2 weeks ago'). (default: None)
    -T {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...], --types {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...]
    Types filter (default: None)
    -D DISTRIBUTION, --distribution DISTRIBUTION
    Distribution filter (default: None)
    -S SOURCE, --source SOURCE
    Source package name filter (default: None)
    -V SOURCE_VERSION, --source-version SOURCE_VERSION
    Version filter (default: None)
    -M MINIMAL_VERSION, --minimal-version MINIMAL_VERSION
    Minimal version filter (default: None)
    -E {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...], --exit-on {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...]
    Types to exit on (default: None)
    -F {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...], --fail-on {REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} [{REJECTED,BUILDING,BUILT,PACKAGING,INSTALLED,FAILED,MIGRATED,REMOVED} ...]
    Types to (return and) fail on (default: None)

    NAME

    drs_checkthredds - Check metadata in a THREDDS catalog is consistent with the DRS

    DESCRIPTION

    Things to check:

    1. All DRS components set as properties and validate with drslib 2. drs_id is consistent with properties 3. version is a date 4. dataset urlPath is consistent with the DRS directory structure. 5. Checksums are present and the right format (NOT 'MD5:...') 6. tracking_id is present 7. Check product assignement is right.

    Currently implemented: 1-3

    VERSION

    3.0.3.p1

    BUGS


    No Known bugs. Please report bugs using reportbug on Debian systems.

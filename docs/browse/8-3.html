Unless separately invoked it is started on demand from smbd or winbind and serves DCERPC only over named pipes (np) as a helper process. This will be the standard setup for most installations (standalone/member server/AD server) unless they modify their startup scripts. Note in when Samba is configured as an Active Directory Domain controller the samba process that invokes smbd will still provide its normal DCERPC services, not samba-dcerpcd. When separately invoked by system startup scripts or a a daemon, the global smb.conf option m[blue]rpc start on demand helpers = falsem[] MUST be set to allow samba-dcerpcd to start standalone.

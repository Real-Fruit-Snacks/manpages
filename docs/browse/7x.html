    section 7x — Overviews & conventions

    • afterstep_faq(7x) This document is an ever growing set of questions, statements, ideas and complaints about AfterStep version 2.0
    • Align(7x) Afterstep Align
    • AnimateTypes(7x) animation modes used in the Animate module's config
    • ASDatabase(7x) defines how windows should be treated by AfterStep depending on its name
    • ASDatabaseEntry(7x) to be used in entry of AfterStep database
    • asimagexml(7x) XML schema to be used for scripting image manipulation by AfterStep and ascompose libAfterImage/asimagexml
    • AutoExec(7x) functions to be executed by AfterStep on startup/shutdown
    • Base(7x) essential AfterStep configuration options
    • Bevel(7x) Afterstep Bevel
    • ColorScheme(7x) defines color values for standard set of internal color names, to be used in other configuration files
    • Feel(7x) AfterStep feel configuration
    • FeelWindowBox(7x) defines placement policy for certain type of windows, based on desk, window attributes, window size, etc. Useful for xinerama configurations, where windows should not be placed in between screens.
    • Functions(7x) built in AfterStep functions
    • Gravity(7x) Afterstep Gravity
    • Look(7x) AfterStep look
    • MyBackground(7x) details how background is to be rendered
    • MyFrame(7x) defines how AfterStep should construct window's frame
    • MyStyle(7x) defines combination of color, font, style, background to be used together
    • PagerDecorations(7x) Visual attributes of the Pager window
    • Placement(7x) AfterStep supports several different window placement policies. Some of them designed to fill free space, and some allowing for windows to be placed on top of others.
    • SupportedHints(7x) Afterstep SupportedHints
    • TbarLayout(7x) Afterstep TbarLayout
    • WharfSounds(7x) types of events for which sound could be played by Wharf
